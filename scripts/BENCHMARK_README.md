# FTD Pipeline Benchmark Results — How to Read

## Hardware

All results below are from a **GH200 480GB** (SM 9.0, 132 SMs, Hopper).

## What Was Benchmarked

Three DSA-2000 Chronoscope pipeline stages at Band 2 production dimensions:

| Pipeline | Full Name | Computation | Key Dimensions |
|----------|-----------|-------------|----------------|
| **VoltBF** | Voltage Beamformer | FP8 complex GEMM × 2 pols + time integrate + corner turn | M=n_time, N=n_beam=4096, K=n_ant=1664, batch=n_ch |
| **VisBF** | Visibility Beamformer | FP8 complex HERK + pol reduce + 2D FP16 FFT imaging + beam extract | N=n_ant, K=n_time/n_time_inner, Ng=4096, n_beam=6M |
| **Dedisp** | Dedispersion (FDD) | Frequency-domain dedispersion via batched GEMM (CuBLAS FP32 vs CUTLASS FP8) | M=Nt_complex, N=n_dm, K=n_ch, batch=n_beam |

Each benchmark runs 3 warmup + 10 timed iterations and reports min/mean/std per stage.

## Benchmark Name Reference

Benchmark labels follow the pattern `<suite>_<config>`. The table below lists
every label generated by `run_benchmarks.sh` and `run_nsys_profile.sh`.

### VoltBF labels

| Label | Meaning |
|-------|---------|
| `voltbf_1ch` .. `voltbf_32ch` | Standard VoltBF with `{1,2,4,8,16,32}` frequency channels (batch dimension), n_beam=4096 |
| `voltbf_M4096` .. `voltbf_M16384` | M-dimension sweep: n_time=`{4096,6144,...,16384}`, n_beam=4096, n_ch=1 (single batch). Tests GPU saturation at large M. |
| `voltbf_short_unfused` | Short-integration baseline: n_time=8, no channel fusion, n_ch=200 (matched batch count for fair comparison) |
| `voltbf_short_cf2` .. `voltbf_short_cf8` | Short-integration with channel fusion: `cf{N}` = ch_fuse=N (M = n_time × ch_fuse) |
| `voltbf_short_cf8_nb2` .. `voltbf_short_cf8_nb16` | Short-integration with ch_fuse=8 + payload batching: `nb{N}` = N payloads batched (M = n_time × ch_fuse × n_batch) |
| `voltbf_1ch_t8` | Quick nsys profile config: 1 channel, n_time=8 |
| `voltbf_8ch_t8` | Quick nsys profile config: 8 channels, n_time=8 |
| `voltbf_prod_t128` | Production-scale nsys profile: n_ch=64, n_time=128 |
| `voltbf_prod_t512` | Production-scale nsys profile: n_ch=64, n_time=512 |

### VisBF labels

| Label | Meaning |
|-------|---------|
| `visbf_1ch_K64` .. `visbf_1ch_K256` | Single channel, K sweep: `K{N}` = HERK K dimension (= n_time / n_time_inner) |
| `visbf_1ch_K128` | Single channel, K=128 (production default) |
| `visbf_2ch_K128` .. `visbf_32ch_K128` | Batch sweep at K=128: `{N}ch` = N frequency channels (batch = n_ch × 2 × n_time_inner) |
| `visbf_1ch_K128_pipe` | Pipelined visibility benchmark (test binary only) |
| `visbf_prod_K64` | Production-scale nsys profile: n_ch=128, K=64, n_beam=6M |
| `visbf_prod_K128` | Production-scale nsys profile: n_ch=128, K=128, n_beam=6M |
| `visbf_128ch_K128` | Large channel count nsys profile (test binary, Ng=64) |

### Dedisp labels

| Label | Meaning |
|-------|---------|
| `dedisp_1600ch_2000dm_b64` | Production FDD: 1600 channels, 2000 DM trials, 64 beams (batch) |
| `dedisp_1600ch_2000dm_b16` .. `dedisp_1600ch_2000dm_b256` | Batch sweep: `b{N}` = N beams |
| `dedisp_1600ch_512dm_b64`, `dedisp_1600ch_1000dm_b64` | DM sweep: `{N}dm` = N DM trials |
| `dedisp_1600ch_2000dm_b64_pipe` | Pipelined dedisp: n_payloads=8, pipeline=2 (CUTLASS only) |

### Label abbreviation glossary

| Abbreviation | Meaning |
|-------------|---------|
| `ch` | Frequency channels (n_ch). Determines batch count for VoltBF/VisBF. |
| `K` | HERK K dimension. For VisBF: K = n_time / n_time_inner. |
| `cf` | Channel fusion factor (ch_fuse). Stacks consecutive channels along GEMM M. |
| `nb` | Payload batch count (n_batch/n_payloads). Stacks consecutive payloads along GEMM M. |
| `b` | Beam/batch count. For Dedisp: n_beam = GEMM batch dimension. |
| `dm` | Dispersion measure trials (n_dm). GEMM N dimension for Dedisp. |
| `t` | Time samples (n_time). Used in nsys profile labels. |
| `pipe` | Pipelined execution mode (double-buffered, overlapping compute and data transfer). |
| `prod` | Production-scale parameters (large n_ch, n_beam=6M for VisBF). |
| `short` | Short-integration mode (n_time=8, small M). |
| `ms` | Multi-stream execution (in streams column). |

## How to Read the Summary CSV

```
benchmark, suite, n_ant, n_beam, n_ch, n_time, K, batch, n_dm, ch_fuse, n_batch, total_mean_ms, throughput_hz, beams_per_ch_sec, tflops, speedup, streams
```

| Column | Full Name | Meaning |
|--------|-----------|---------|
| `benchmark` | Benchmark label | Config identifier (see Benchmark Name Reference above) |
| `suite` | Pipeline suite | Which pipeline: `voltbf`, `visbf`, or `dedisp` |
| `n_ant` | Number of antennas | Array elements (1664 for DSA-2000, padded from 1651 for MMA alignment) |
| `n_beam` | Number of beams | Output beams: N_beam for VoltBF (GEMM N dim), 6000000 for VisBF (image pixels extracted), batch dim for Dedisp |
| `n_ch` | Number of frequency channels | Frequency channels per payload. Determines HERK/GEMM batch count for VoltBF and VisBF. Equals GEMM K for Dedisp. |
| `n_time` | Number of time samples | Time samples per payload. VoltBF: GEMM M dimension (or M = n_time × ch_fuse × n_batch). VisBF: divided by n_time_inner to get HERK K. |
| `K` | GEMM/HERK K dimension | The reduction dimension. VisBF: K = n_time / n_time_inner. VoltBF: K = n_ant. Dedisp: K = n_ch. |
| `batch` | GEMM/HERK batch count | Number of independent GEMM/HERK problems. VisBF: n_ch × 2 × n_time_inner. VoltBF: n_ch (or n_ch / ch_fuse when fused). Dedisp: n_beam. |
| `n_dm` | Number of DM trials | Dispersion measure trial count (Dedisp only). GEMM N dimension. |
| `ch_fuse` | Channel fusion factor | Number of consecutive channels stacked along M (VoltBF fused only). Increases effective M by ch_fuse×. |
| `n_batch` | Payload batch count | Number of consecutive time payloads batched into one GEMM call (VoltBF fused only). Increases effective M by n_batch×. |
| `total_mean_ms` | Total mean time (ms) | Mean wall-clock time per payload across all timed iterations, in milliseconds |
| `throughput_hz` | Throughput (payloads/sec) | Payloads processed per second (= 1000 / total_mean_ms) |
| `beams_per_ch_sec` | Beams per channel per second | Science-relevant throughput: n_beam × payloads/sec. Measures how many beam-channel products are computed per second. |
| `tflops` | Effective TFLOPS | Tera floating-point operations per second. Uses complex FP8 FLOP counting: 8 × M × N × K × batch (4 real sub-GEMMs × 2 FMA ops each). |
| `speedup` | CUTLASS/CuBLAS speedup | Ratio of CuBLAS FP32 time to CUTLASS FP8 time (Dedisp only). Values > 1 mean CUTLASS is faster. |
| `streams` | Execution mode | `single` (default sequential), `pipelined` (double-buffered overlap), or `multi` (multi-stream concurrent) |

## How to Read the Per-Stage Breakdown

```
benchmark, suite, stage, min_ms, mean_ms, std_ms, pct_total
```

| Column | Full Name | Meaning |
|--------|-----------|---------|
| `benchmark` | Benchmark label | Same label as in summary CSV |
| `suite` | Pipeline suite | `voltbf`, `visbf`, or `dedisp` |
| `stage` | Pipeline stage name | Kernel or stage identifier (see tables below) |
| `min_ms` | Minimum time (ms) | Fastest iteration across all runs |
| `mean_ms` | Mean time (ms) | Average time across all timed iterations |
| `std_ms` | Standard deviation (ms) | Run-to-run timing variability |
| `pct_total` | Percentage of total | This stage's mean time as a percentage of total pipeline time |

### VoltBF stages
| Stage | What it does |
|-------|-------------|
| `qc_cast` | INT4 QC (quantised complex) input transpose + polarisation split (+ optional FP8 cast in fused path) |
| `gemm_pol0` | FP8 complex GEMM for polarisation 0 (beta=0, overwrite output) |
| `gemm_pol1` | FP8 complex GEMM for polarisation 1 (beta=1, accumulate Stokes I) |
| `time_integrate` | Time power sum: reduce n_tps consecutive time samples and quantise to uint8 |
| `corner_turn` | Output transpose from (n_ch, n_beam) to (n_beam, n_ch) layout |

### VisBF stages
| Stage | What it does |
|-------|-------------|
| `herk` | FP8 complex Hermitian rank-K update → packed triangle visibilities |
| `pol_reduce` | Sum 2 polarisation visibilities per frequency channel |
| `img_scatter` | Pillbox gridding: scatter visibilities onto Ng×Ng UV plane |
| `img_fft` | Batched 2D FP16 inverse FFT via cufftXt (UV plane → image) |
| `img_beam` | Extract n_beam pixel intensities from the image cube |
| `corner_turn` | Output transpose from (n_ch, n_beam) to (n_beam, n_ch) layout |

### Dedisp stages
| Stage | What it does |
|-------|-------------|
| `CuBLAS_FP32` | Frequency-domain dedispersion via cuBLAS FP32 complex GEMM (baseline) |
| `CUTLASS_FP8` | Frequency-domain dedispersion via CUTLASS FP8 complex GEMM (optimised path) |

### Test binary stages (fallback when bench_pipeline is unavailable)
| Stage | What it does |
|-------|-------------|
| `H2D` | Host-to-device data transfer |
| `Compute` | GPU compute (all kernels) |
| `D2H` | Device-to-host data transfer |
| `Preamble` | Pipeline setup (allocations, weight preparation) |
| `Epilogue` | Pipeline teardown and result validation |

## How to Read the Pipelined CSV

```
benchmark, mode, total_ms, per_payload_ms, throughput_hz, speedup
```

| Column | Full Name | Meaning |
|--------|-----------|---------|
| `benchmark` | Benchmark label | Config identifier (always a `*_pipe` label) |
| `mode` | Execution mode | `Serial` / `Sequential` (one at a time) or `Pipelined` (double-buffered overlap) |
| `total_ms` | Total wall time (ms) | Total elapsed time for all payloads |
| `per_payload_ms` | Per-payload time (ms) | Effective time per payload (= total / n_payloads) |
| `throughput_hz` | Throughput (payloads/sec) | 1000 / per_payload_ms |
| `speedup` | Pipeline speedup | Ratio of serial time to pipelined time. Values > 1 mean pipelining helps. |

## Key Observations

### VoltBF
- **GEMM dominates** (~88% of total time, split across 2 pols)
- **Scales linearly** with n_ch (batch dimension): 1ch=7.3ms, 32ch=229ms
- **Sustained 67-69 TFLOPS** across all batch sizes — GPU fully saturated
- `gemm_pol1` is ~14% slower than `gemm_pol0` (beta=1 accumulation vs beta=0)

### VisBF
- **FFT dominates** (~35-50% of total), followed by beam extraction (~20-28%)
- HERK is only 11-23% of total — imaging is the bottleneck
- FFT time scales linearly with channels: 0.73ms/ch (from 1ch to 32ch trend)
- K sweep (64→256) shows HERK scaling but FFT is constant (same grid size)

### Dedisp
- **CUTLASS FP8 is 2.7-5.2× faster** than CuBLAS FP32
- Speedup increases with batch size (better GPU utilization)
- Per-beam time decreases with batch: b16=0.24ms/beam → b256=0.058ms/beam
- At b256: 28.4 TFLOPS, 17.3k beams/sec

## Dedisp Memory Note

The FDD pipeline pads `n_time` to the next power of 2 (plus dispersion delay bins)
and allocates a shift matrix of size `n_ch × n_dm × Nt_complex`. At production
n_time=9000, Nt_padded=32768, making the shift matrix alone 210 GB — infeasible.
Benchmarks use n_time=256 (Nt_padded=1024, shift matrix=6.6 GB), representing
one processing chunk. The real pipeline would process the full 9000 time samples
in sequential chunks.

## Reproducing

```bash
# Full suite
bash scripts/run_benchmarks.sh --build-dir /data/dmhowart/pipeline/build --runs 10

# Single suite
bash scripts/run_benchmarks.sh --build-dir /data/dmhowart/pipeline/build --suite voltbf

# Parse results
bash scripts/parse_benchmarks.sh bench_results/
```
